# **App Name**: PulseGuard

## Core Features:nnn

- Patient Account Creation: Doctors can create and manage patient accounts, including personal details, medical history, and sensor data.
- Doctor Account Management: Allow doctors to manage their profiles, including name, age, picture, and license information.
- Sensor Data Ingestion: Collect and store real-time data from patient sensors (O2 levels, room temperature, patient temperature, heart rate, room humidity).
- Real-time Monitoring Dashboard: Display real-time sensor data in an easy-to-understand format for doctors.
- Alert System: Generate alerts based on abnormal sensor readings, customizable by doctors.
- Patient History: Store and display historical sensor data and medical history for each patient.
- Predictive Anomaly Detection: Use a generative AI tool to learn patterns of physiological data, in order to proactively recognize when a patient's condition could be starting to deteriorate. Generate alerts if the anomaly level rises above a set level.

## Style Guidelines:

- Primary color: Deep blue (#2962FF) to evoke a sense of trust and stability, aligning with the app's purpose in healthcare.
- Background color: Light gray (#F0F4F8) to provide a clean and calm interface, ensuring data visibility and reducing visual stress.
- Accent color: Vibrant green (#3BB78F) for positive feedback and critical alerts to quickly draw attention to important information.
- Body and headline font: 'PT Sans' (sans-serif) for clear and accessible readability across all data displays and user interface elements.
- Use simple, clear icons to represent different sensor data types and actions. Icons should be monochromatic and align with the primary color.
- Maintain a clean, well-organized layout with clear visual hierarchy to ensure ease of navigation and quick access to essential patient information.
- Use subtle transitions and animations for loading data and displaying alerts to provide feedback without being distracting.
